Building an Inclusive Web

My commitment is to create digital experiences that are accessible and usable for everyone, regardless of ability or the technology they use.

My Approach to Accessibility

I believe accessibility is not a feature, but a fundamental part of the design and development process. By integrating accessibility from the start, we can build more robust, usable, and innovative products for a wider audience.

Core Accessibility Skills

  • Semantic HTML: Using landmarks and proper HTML5 structure to create a solid foundation for assistive technologies.
  • ARIA Implementation: Applying WAI-ARIA roles and properties to enhance the accessibility of dynamic and custom components.
  • Keyboard Navigation: Ensuring every interactive element is fully operable with a keyboard alone, with logical focus order.
  • WCAG Standards: Adhering to the Web Content Accessibility Guidelines (WCAG) 2.2 at AA and AAA levels.

Tools & Testing

A combination of automated tools and manual testing is crucial for effective accessibility auditing. My toolkit includes:

  • Screen Readers: In-depth manual testing using NVDA on Windows to simulate the user experience of visually impaired individuals.
  • Developer Tools: Utilizing browser dev tools to inspect the accessibility tree, color contrast, and element properties.
  • Manual Audits: Performing keyboard-only navigation checks and assessing visual focus indicators.
